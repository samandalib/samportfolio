Locatify

Indoor Navigation | AR Mobile app

About

We all had this experience of being in a new place and felt that we needed someone to help us find out where to go or what to do next. Our initial concept came from our personal experiences. we visit new buildings and new places very often. It’s common that we find ourselves unable to find the room we're looking for inside a building. In this project we propose a mobile application prototype that aims at making the process of finding a room in a building easier.

MY ROLES
 Ideation, Prototype Design, Usability Test

MY TAKE OVERS
 Understanding the challenges of using AR applications, and the ways that can make them easier to be used by users in their routine life

FUTURE CHALLENGE
 Creating easy and purposeful AR content by non-technical users

The Challenge

How might we help people find their way inside a building complex without making them to understand building layout or asking others?

User Research
Interview

Based on survey results, we interviewed four students from UT Austin to learn more about their experience in finding places on campus. During the interview, we asked open-ended questions to elicit interviewees’ stories, thoughts, and actions.

User Journey Map
Based on the insights we got from the affinity diagram, we created a user journey map. We identified users’ pain points and our product’s opportunities in the entire journey.


The indoor navigation application in the first iteration of the design cycle had two parts. The first part, “Find AR content” was for users to find information about a place. The second part, “Create AR content” was for users to create AR content in the space. Once the user taps the first button, the app will ask what the user is looking for, and then users can type in the room code.

Path Vs. Target
One of the biggest differences between indoor and outdoor navigation is that all the objects are closer to the user when they are inside of a building. Hence, we believes that the best solution for interior navigation is highlighting targets for users instead of showing a path on the ground. The app highlights objects that users need to pass by with instructions in a box.

Decisions
based on the findings at this stage, we made the following decisions:

Based on the feedback and findings we got from the lo-fi testing, we iterated the prototype into medium-fidelity. The first change is that a voice command feature is added to the application. So the user can select if they want to talk to the system or use text input. If user chooses voice assistance, it will then ask users the room number and then confirms the number by repeating it. If the user chooses to use text, they can simply put the room code into the box, and the system can identify the building code and room number.

Features and Functions
We moved the text bar from the bottom to the upper, changed the rectangle into gradient color to highlight targets. When the user’s phone is not toward a target, a blue arrow notifies the user to change the direction. If the highlighted target is a door, the text box reminds the user to try Reroute option from menu. We also added a big icon in front of the destination.
