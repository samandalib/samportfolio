Application Stack: a bundle of software that runs everything from the frontend to the backend, from database to server administration to rendering the website. There are some common bundles. The most Famous being the LAMP stack in which
L = Linux (as the administrative software)
A = Apache (as the web server software)

Now that I understood the Stack of my application. I followed the instruction in this Youtube video to deploy my application on Heroku. However the work was not as smooth as I expected. Soon I ran into a problem and it was:

I searched for the problem on Stackoverflow and found that the command npm audit fix might help to resolve the issue. The good news was that the build folder was now created in the Client directory but the bad news was that I was still unable to deploy the application on Heroku. After searching in the google and Stackoverflow and reading the community questions. I found the issue. It was very interesting though:
in the "package.json" file on the server side I had added the following command at "scripts" part :

But the problem was that as a habit of practice, I put a comma (,) at the end of the line (like what I do after the last element an array / object in JavaScript) and it was the cause of the problem. I removed it and YES! deployed on Heroku successfully.

Step 3: Connecting to Linux instance

Step 4: Transferring files to the Linux instance
After the connection to the Linux instance command line the files can be transferred to the server with a single command. For example if you want to transfer a file with name "sample_file.txt" from the "C:\" rout on the windows to the "EC2-user" home directory on the Amazon Linux instance, the command line will be:
pscp -i c:\path\my-key-pair.ppk c:\path\sample_file.txt ec2-user @ public_dns:/home/ec2-user/sample_file.txt

I started Learning the MERN stack from the Node.js part. I found an interesting course on edx.org with title "Introduction to Node.js". It started with basics of installing and setting a Node.js environment. The main takeaways of this course for me were as follows:
  • Node is an interpreted language not a compiled one: that it is not as fast as comiled languages like C#, C++, etc. but it provides the opportunity to make changes in the code and see the results instantly
  • There are three main ways to launch a Node code: REPL, Eval Option, and Launching code from a file
  • Node.js and browser JavaScript differ when it comes to globals. Developers should be familiar with these keywords: Process, Global, module.exports and exports
During this course I learned about the following topics:
  • Node.js Globals
  • Node.js Process
  • Command-line Arguments
  • "require" and "Module.exports"
  • Core Modules like "fs", "path", "http"
  • Event emmiters

Learning Node.js requires getting familiar with "npm" (Node package Manager) as well. Npm consists of three components:
  • Website: www.npmjs.com
  • command-line tool: npm
  • Registries: public and private

We need either the package.json file or "node_modules" folder to install modules locally. The best thing about npm is that it keeps all the dependencies local.
To install a module locally we use either npm install module-name or npm i module-name
To install a module globally, we use either npm install --global module-name or npm i -g module-name
To remove an npm module locally, use npm rm module-name
To remove a global moduel, use npm rm module-name -g

Express.js

Express.js is a web framework based on the core Node.js "http" module and connect components. The components are called "middleware" and they are the cornerstones of the framework philosophy "configuration over convention". i.e. Express systems are highly configurable, which allows developers to freely pick whatever libraries they need for a particular project. Some of the benefits of express include:
  • Deliver static content
  • Modularize business logic
  • construct an API
  • Connect to various data source (with additional plugins)
  • Write less code (vs. http)
  • Validate data (with additional plugins)
The main Express app file is typically named app.js, server.js or index.js. It has the following sections:
  1. Imports
  2. Instantiations
  3. Configurations
  4. Middleware
  5. Routes
  6. Error handlers
  7. Server Bootup or server export

MongoDB

MongoDB is a NoSQL database. A good reason to use NoSQL databases is that, because they are schema-less, they are perfect for prototyping and for agile iterations. It is also a document store NoSQL databese (as opposed to key-value and wide-column NoSQL databases). It is the most mature and dependable NoSQL database available thus far.
MongoDB uses JavaScript. So all JavaScript and Node commands, classes and methods are a fair game in the Mongo console. In this section of learning, I learned the following skills:
  • connecting to MongoDB
  • Updating documents
  • Removing Documents
  • Finding Documents
  • Coding CRUD (Create, Read, Update, Delete) REST API with Node, Express and MongoDB
  • Mongoose library: a fully developed object-document mapping (ODM) library for Node.js and MongoDB
